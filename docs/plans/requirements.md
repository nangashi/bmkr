# 模擬ECサイト 要件定義

## 目的

AI駆動開発の練習を目的とした模擬ECサイトの構築。
本サービスはローカル環境での実行のみを想定し、外部サービスへのデプロイは行わない。

## システム全体像

モノレポ内に以下の4サービスをマイクロサービスとして配置する。

```
[ブラウザ] ──────────────────────────────────────────────┐
    │                                                    │
    │ ECサイト画面                       管理画面（商品管理・顧客管理）
    │                                                    │
    ▼                                                    ▼
  [BFF] ─────┬──────────┬──────────┐   [商品管理UI]  [顧客管理UI]
             │          │          │        │              │
             ▼          ▼          ▼        ▼              ▼
         [ECサイト] [商品管理API] [顧客管理API]
             │          │          │
             ▼          ▼          ▼
           [DB]       [DB]       [DB]
```

- ECサイトの画面はBFFを経由してバックエンドにアクセスする
- 商品管理・顧客管理の管理画面は各サービスが直接提供する

- サービス間はAPIで連携する
- 各サービスは独立してデプロイ可能
- デプロイ先は同一サービス上でもよい
- 技術スタック・デプロイ先は別途ADRで決定する

## サービス別要件

### 1. BFF（Backend for Frontend）

ECサイトのフロントエンドに対するAPIゲートウェイとして機能する。

- 各マイクロサービスのAPIを集約し、フロントエンド向けに最適化したAPIを提供する
- 認証状態の管理・検証を行う
- フロントエンドが複数サービスを直接呼ばずにBFFを経由する

### 2. ECサイト（ストアフロント）

一般ユーザー向けのECサイト。画面とドメインロジック（カート・注文）を持つ。
画面はBFFを経由してアクセスされる。

**認証**
- ID/パスワード認証によるログイン・ログアウト（BFF経由）
- 未ログイン時は商品閲覧のみ可能

**商品閲覧**
- 商品一覧の表示（商品管理APIから取得）
- 商品詳細の表示（商品管理APIから取得）
- 商品画像は扱わない（画面上は No Image プレースホルダーを表示）

**カート**
- カートへの商品追加・削除・数量変更
- カート内容の確認

**注文**
- カート内容をもとに注文を確定する
- 注文時に商品管理APIへ在庫引き当てを依頼する
- 注文履歴の閲覧

### 3. 商品管理

管理者向けの商品管理システム。管理画面とAPIを持つ。

**認証**
- 管理画面へのアクセスに認証が必要（ID/パスワード認証）

**商品CRUD**
- 商品の登録・編集・削除
- 商品情報: 商品名、説明、価格、在庫数
- 商品画像は扱わない

**商品一覧**
- 商品の一覧表示

**在庫管理**
- 在庫数の更新
- 注文時の在庫引き当てAPI（ECサイトからの連携）

### 4. 顧客管理

管理者向けの顧客管理システム。管理画面とAPIを持つ。

**認証**
- 管理画面へのアクセスに認証が必要（ID/パスワード認証）

**顧客一覧**
- 顧客の一覧表示

**顧客詳細**
- 顧客情報の閲覧（ユーザー登録時に入力された情報: 名前、メールアドレス等）

**注文履歴**
- 顧客ごとの注文履歴の閲覧（ECサイトAPIから取得）

## サービス間連携

| 呼び出し元 | 呼び出し先 | ユースケース |
|-----------|-----------|------------|
| BFF | ECサイト | 商品閲覧、カート操作、注文処理 |
| BFF | 商品管理 | 商品情報の取得（ストアフロント表示用） |
| BFF | 顧客管理 | 顧客情報の取得・登録 |
| ECサイト | 商品管理 | 注文時の在庫引き当て |
| ECサイト | 顧客管理 | 注文時の顧客情報参照 |

- Docker Composeによる全サービスの統合起動
- 各サービスは個別に起動・開発も可能
