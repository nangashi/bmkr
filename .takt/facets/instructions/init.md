issue-plan で作成された実装計画をもとに、実装準備を行う。

## 手順

### 1. Issue 情報の読み込み

タスク指示（`{task}`）から Issue 番号を特定する。
`gh issue view {番号}` と `gh issue view {番号} --comments` で Issue 本文とコメントを読み込む。

### 2. 計画の検出

Issue コメントから `<!-- issue-plan:plan:done -->` マーカーを検索する。
計画が見つからない場合は「計画が見つからない」と報告して終了する。

### 3. 開発準備

worktree では gitignore 対象のファイルが存在しないため、以下を実行する:

```bash
pnpm install && just generate
```

### 4. 実行レベルの判定

計画の内容を分析し、実装に必要なフェーズを判定する:

| 計画の内容 | 判定 | 次のムーブメント |
|-----------|------|----------------|
| 新しい型・インターフェース・関数の定義が必要 | 「新しい型・インターフェースの定義が必要」 | interface_design |
| 既存コードの修正で新しいロジック分岐がある | 「既存コード修正で新規ロジックあり」 | interface_design |
| 機械的な置換・削除のみ | 「機械的置換のみ」 | implement |

### 5. レポート出力

以下の内容を含むレポートを出力する:

```markdown
# 初期化コンテキスト

## Issue 番号
{番号}

## Issue 本文
{Issue 本文全文}

## 実装計画
{計画コメントの全文}

## 方針（あれば）
{方針検討コメントの全文。なければ「なし」}

## 実行レベル判定
{判定結果と根拠}
```
