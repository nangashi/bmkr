# Review: セキュリティ

## Overview
エラーレスポンスの情報漏洩、入力バリデーション、secrets 非埋め込みをレビューする。

## Step 1: Analysis

- HTTP / gRPC レスポンスを構築する箇所
- ユーザー入力を受け取る箇所
- 設定値・シークレットの読み込み箇所
- SQL クエリやファイルパス構築箇所

## Step 2: Findings

### 指摘とする基準

- 内部実装の詳細を含むエラーレスポンス
- 入力バリデーション不足
- シークレットのハードコード
- 危険な文字列連結クエリやパス構築

### 指摘としない基準

- sqlc 生成クエリ
- テストコード内の固定値
- 開発用サンプル設定値

セキュリティ
