# Review: 型設計

## Overview
公開 I/F の粒度、構造体フィールド可視性、不変条件の型表現が適切かをレビューする。

## Step 1: Analysis

- 新規・変更された struct 定義
- 新規・変更された interface 定義
- 関数シグネチャ
- `contract.md` に記載された公開 I/F

## Step 2: Findings

### 指摘とする基準

- interface が過剰に広い
- 不変条件を壊しやすい公開 struct になっている
- 契約で要求する区別が型に表れていない
- テスト差し替えが必要なのに具象依存が強い

### 指摘としない基準

- フレームワークが要求する型への準拠
- 既存コードベースで確立されたパターン
- 自動生成コード

### perspective ラベル
型設計
