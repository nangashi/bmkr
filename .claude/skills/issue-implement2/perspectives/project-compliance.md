# Review: プロジェクト準拠

## Overview
ルール準拠、ADR/ガイド準拠、既存パターンとの一貫性、命名規則をレビューする。

## Step 1: Analysis

- `.claude/rules/` の適用対象ルール
- 関連 ADR
- 関連 guides
- 変更対象と同一パッケージの既存コード

## Step 2: Findings

### 指摘とする基準

- ルール違反
- ADR に反する実装
- guide に反する実装
- 既存コードと乖離した命名や構成

### 指摘としない基準

- 前例のない領域での新しいパターン
- テストコードの細かな構成差
- 自動生成コード

### perspective ラベル
プロジェクト準拠
