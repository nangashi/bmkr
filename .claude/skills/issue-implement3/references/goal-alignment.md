# ゴール整合性

受け入れ条件のトレーサビリティとスコープ逸脱をチェックする。

---

## 実行

Issue 番号を渡して Sonnet サブエージェント（`agents/goal-alignment.md`）を起動する。

サブエージェントは以下を検証する:

### 1. 受け入れ条件トレーサビリティ

Issue の受け入れ条件（計画コメントに記載）を 1 つずつ確認し、以下に分類する:

- **充足**: コード変更またはテストで確認できる
- **未対応**: 対応するコード変更が見つからない
- **要確認**: 部分的に対応しているが完全ではない可能性

### 2. スコープ逸脱チェック

Issue の「スコープ外」に記載された内容に触れる変更がないか確認する。

### 3. Invariants 検証（L3 フローで実行されている場合のみ）

L3 フローで contract.md が作成されている場合に実行する（L1/L2 では contract.md が存在しないためスキップされる）。Invariants セクションに列挙された不変条件が全て維持されていることを確認する:

- `just test` の通過確認（品質ゲートの修正による退行を検出するため再実行する）
- Invariants リストと現在のコードを照合し、不変条件が壊れていないことを確認

---

## 結果の処理

- **全て充足 + Invariants OK** → PR 作成に進む
- **未対応あり or Invariants 違反** → 実装フェーズに戻る（最大 2 回。2 回目以降は品質ゲートをスキップしゴール整合性のみ再実行）
- **2 回戻っても解決しない** → ユーザーに報告し判断を仰ぐ
