# 振り返り

PR 作成後、または ABORT 後に実行する。

---

## 実行

Issue 番号を渡して Sonnet サブエージェント（`agents/retrospective.md`）を起動する。サブエージェントは `.output/` 内のファイルを自己取得して分析する。

## 共通観点（全レベル）

1. **空振り Phase の検出**: レビューで指摘ゼロだった Phase。フローの最適化余地を示す
2. **手戻りの検出**: Phase 間の逆行（ゴール整合性→実装、Escalation→契約修正 等）の回数と原因
3. **レベル分類の妥当性**: 実際の変更内容とレベルが適切だったか

## レベル固有の観点

各フローファイル（flow-L1/L2/L3.md）の振り返りセクションに記載。

## 出力

改善提案をメインエージェントに返す。メインエージェントはユーザーに提示し、承認された改善を実行するか改善 Issue を作成する。
