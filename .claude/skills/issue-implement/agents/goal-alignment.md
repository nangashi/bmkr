# ゴール整合性レビュー — サブエージェント詳細手順

実装結果が Issue の受け入れ条件を満たしているか、スコープを逸脱していないかを検証する。

---

## 前提情報の取得

- `gh issue view {issue_number}` で Issue 本文（受け入れ条件・スコープ外）を読む
- `gh issue view {issue_number} --comments` で計画コメント（`<!-- issue-plan:plan:done -->` マーカー付き）を読む
- `git diff main` で実装差分を把握する

---

## トレーサビリティ表の作成

受け入れ条件ごとに、対応するコードとテストを特定し、以下の表を埋める:

| 受け入れ条件 | 対応コード（ファイル:関数） | 対応テスト（ファイル:テスト名） | 状態 |
|-------------|--------------------------|------------------------------|------|
| 条件1 | handler.go:HandleX | handler_test.go:TestX | OK / 未対応 / 要確認 |

対応コード・テストの特定には、差分だけでなく実際のファイルを Read して確認すること。

---

## スコープ逸脱チェック

以下の観点で確認する:

- Issue の「スコープ外」に記載された内容に触れていないか
- 計画にない追加機能・リファクタリングが含まれていないか
- 受け入れ条件の独自解釈がないか

---

## 出力

結果を `.output/issue-implement/{issue_number}/goal-alignment.md` に書き出す。

トレーサビリティ表と、スコープ逸脱チェックの結果を含める。「未対応」「要確認」またはスコープ逸脱がある場合は、その旨を明記する。
